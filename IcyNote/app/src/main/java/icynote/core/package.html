Provides the core interfaces and factories to be used by user code such as the UI or main() as well as some template tests specifying their behaviour.

The core application is where the interactors and boundaries reside.

  • An Interactor is a unit of business logic that performs operations and/or data validation.
  • A Boundary is an interface whose purpose is to isolate the core from its plugins, such as the User Interface (UI) or the Storage implementation (filesystem, database, etc.)
